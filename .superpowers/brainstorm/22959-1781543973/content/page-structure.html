Which page structure fits the goal?

Educate first, then activate into Model Lab — where does each section live?

① HERO — Big promise + "Try Model Lab" CTA
② LIVE PREVIEW — Embedded Model Lab teaser (interactive)
③ FEATURES — What makes Nyquiste different (3–4 cards)
④ HOW IT WORKS — 3 steps
⑤ CTA — Sign up / pricing

A — Demo-Led

Model Lab preview appears early, right after the hook. Visitors see the product in action before reading any feature copy. High activation intent.

① HERO — Big promise + "Explore Models" CTA
② PROBLEM / PAIN — Why global AI is hard (3 pain points)
③ SOLUTION — How Nyquiste solves it (features grid)
④ MODEL LAB CTA — Prominent callout section + preview shot
⑤ PRICING TIERS — Inline pricing overview
⑥ FINAL CTA — Sign up

B — Story-Led

Tells the classic problem → solution narrative before asking for anything. Model Lab gets its own dedicated showcase section mid-page. Educates before activating.

① HERO — Headline + two CTAs: "Try Model Lab" · "See Pricing"
② SOCIAL PROOF / LOGOS — Supported models (OpenAI, Claude, Gemini…)
③ FEATURES — Bento grid of differentiators
④ MODEL LAB SPOTLIGHT — Full-width showcase + interactive hint
⑤ HOW IT WORKS — 3 steps
⑥ CTA — Get started / pricing

C — Confidence-Led

Hero leads, then immediately shows model logos to build credibility. Features bento establishes trust. Model Lab gets a full-width spotlight. The most balanced structure.
